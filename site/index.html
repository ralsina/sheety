What Even Is This?

Sheety is a tool that takes a spreadsheet definition (YAML or Excel), compiles it to a Crystal program, and builds a standalone binary. Yes, really. We asked ourselves "what if spreadsheets were compiled binaries?" and then we just kept going.

Each spreadsheet becomes its own executable with a terminal UI (TUI). No dependencies. No runtime. Just pure, unadulterated spreadsheet power in a binary.

Features (Some of Which Are Actual Features)

üöÄ Hash-Based Caching

Smart caching means re-running the same spreadsheet is ~400x faster. The first run compiles, subsequent runs are instant. It's almost like we know what we're doing.

üìÅ Excel Import

Convert .xlsx files to YAML and compile them. Because who doesn't want their Excel sheets as binaries?

üé® Terminal UI

Interactive TUI for editing cells and formulas. Navigate with arrow keys, edit formulas, press Q to exit. It's like the 1980s, but with hash-based caching.

üìù Formula Support

Supports common spreadsheet functions: SUM, AVERAGE, IF, VLOOKUP, and more. The formula parser is, dare we say, adequate.

üîÑ Self-Rebuilding

Edit a formula in the TUI and it recompiles itself. The binary rebuilds the binary. It's binaries all the way down.

üíÄ No Dependencies

The resulting binary has zero runtime dependencies. Just run it. No JVM, no Python, no Node.js. Just pure compiled goodness.

How To Use This... Thing

$ shards build
[builds sheety]
$ ./bin/sheety examples/test.yaml
Building /home/user/.local/share/sheety/tmp/a3f5b8c2d9e1f4a6...
Built successfully!
Launching TUI...
[Press Q to exit]

Then run it again:

$ ./bin/sheety examples/test.yaml
Using cached source: /home/user/.local/share/sheety/tmp/a3f5b8c2d9e1f4a6.cr
Using cached binary: /home/user/.local/share/sheety/tmp/a3f5b8c2d9e1f4a6
Built successfully!
[Instant startup ~400x faster]

YAML Format

Define your spreadsheet in YAML:

Sheet1:
  A1:
    value: 10
  A2:
    value: 20
  A3:
    formula: "=SUM(A1:A2)"
  A4:
    formula: "=IF(A3>25, \"Big\", \"Small\")"

Use Cases (We Had To Stretch For These)

  • Embedded Systems: Run spreadsheets on your toaster (ARM support coming maybe)
  • High Performance: Hash-based caching makes opening spreadsheets ridiculously fast
  • Distribution: Send a single binary instead of a fragile Excel file with broken formulas
  • Learning: See how spreadsheet formulas get parsed and compiled to code
  • Because: Why not compile YAML to binaries? Who's going to stop you?

Installation

$ git clone https://github.com/ralsina/sheety
$ cd sheety
$ shards install
$ shards build

Requires Crystal. If you don't have Crystal, install it. If you don't want to install Crystal, well, this whole project was already a stretch anyway.

The "Why" Question

"Why compile spreadsheets to binaries?" you ask, with genuine concern for our mental state.

Look, we don't know. It started as a simple question and escalated. But here we are, with hash-based caching, a TUI, Excel import, and formulas that work. Sometimes the best projects are the ones that make no sense but somehow actually work.

Plus, we added hash-based caching that makes it 400x faster on subsequent runs. So now it's fast and ridiculous. That's a win in our book.
